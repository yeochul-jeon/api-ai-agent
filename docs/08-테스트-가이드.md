## 테스트 실행 방법

### 사전 준비

JDK 21이 설치되어 있어야 합니다:

```bash
java -version
# openjdk version "21.0.x" ...
```

테스트 의존성 (`build.gradle.kts`):
- `spring-boot-starter-test` - JUnit 5 + AssertJ + Mockito 포함
- `junit-platform-launcher` - JUnit 5 실행기

### 테스트 실행

```bash
# 전체 테스트
./gradlew test

# 상세 출력
./gradlew test --info

# 특정 테스트 클래스
./gradlew test --tests "com.apiagent.executor.DuckDbExecutorTest"

# 특정 테스트 메서드
./gradlew test --tests "com.apiagent.executor.DuckDbExecutorTest.extractTables_list_usesName"

# 특정 패키지의 모든 테스트
./gradlew test --tests "com.apiagent.recipe.*"

# 실패한 테스트만 재실행
./gradlew test --rerun

# 테스트 리포트 (build/reports/tests/test/index.html)
./gradlew test && open build/reports/tests/test/index.html
```

### Python 원본과의 차이

| Python | Spring Boot |
|--------|------------|
| `pytest tests/ -v` | `./gradlew test` |
| `pytest -k "recipe"` | `./gradlew test --tests "*Recipe*"` |
| `pytest --lf` | `./gradlew test --rerun` |
| `ruff check` (린터) | 별도 린터 없음 (IDE 의존) |
| `ty check` (타입 체크) | 컴파일 시 자동 (Java는 정적 타입) |
| `pytest-asyncio` | 불필요 (Virtual Threads = 동기 코드) |

### 테스트 파일 조직 패턴

테스트는 JUnit 5 + AssertJ 기반이며, 중첩 클래스(`@Nested`)나 일반 클래스로 그룹화됩니다:

```java
// src/test/java/com/apiagent/mcp/ToolNamingServiceTest.java
class ToolNamingServiceTest {

    private final ToolNamingService service = new ToolNamingService();

    @Test
    void toSnakeCase_convertsCamelCase() {
        assertThat(service.toSnakeCase("API Agent")).isEqualTo("api_agent");
    }

    @Test
    void getToolNamePrefix_extractsMeaningfulParts() {
        assertThat(service.getToolNamePrefix(
            "https://flights-api-qa.internal.example.com/graphql"))
            .isEqualTo("flights_example");
    }

    @Test
    void getToolNamePrefix_nullReturnsDefault() {
        assertThat(service.getToolNamePrefix(null)).isEqualTo("api");
    }
}
```

## 테스트 패턴

### 순수 단위 테스트 (주요 패턴)

이 프로젝트의 대부분의 테스트는 **순수 단위 테스트**입니다. `@SpringBootTest`나 `@DataJpaTest` 같은 슬라이스 테스트는 사용하지 않습니다.

```java
class RecipeNamingTest {

    @Test
    void sanitize_removesSpecialChars() {
        assertThat(RecipeNaming.sanitize("Get Recent Flights!"))
            .isEqualTo("get_recent_flights");
    }

    @Test
    void sanitize_nullReturnsRecipe() {
        assertThat(RecipeNaming.sanitize(null))
            .isEqualTo("recipe");
    }

    @Test
    void deduplicate_appendsSuffix() {
        Set<String> seen = new HashSet<>(Set.of("get_flights"));
        assertThat(RecipeNaming.deduplicate("get_flights", seen, 40))
            .isEqualTo("get_flights_2");
    }
}
```

### AssertJ 사용 패턴

```java
// 기본 검증
assertThat(result).isEqualTo("expected");
assertThat(result).isNotNull();
assertThat(result).isTrue();

// 컬렉션 검증
assertThat(list).hasSize(3);
assertThat(list).contains("item1", "item2");
assertThat(map).containsKey("key");

// 예외 검증
assertThatThrownBy(() -> holder.require())
    .isInstanceOf(IllegalStateException.class)
    .hasMessageContaining("not set");

// 문자열 검증
assertThat(result).startsWith("r_");
assertThat(result).hasSize(40);
```

## Mockito 모킹 패턴

### 기본 패턴: `@Mock` + `@InjectMocks`

외부 의존성이 있는 서비스를 테스트할 때:

```java
@ExtendWith(MockitoExtension.class)
class RestApiClientTest {

    @Mock
    private RestClient restClient;

    @InjectMocks
    private RestApiClient restApiClient;

    @Test
    void execute_getRequest_success() {
        // given
        when(restClient.get()
            .uri(any())
            .retrieve()
            .body(Map.class))
            .thenReturn(Map.of("id", 1, "name", "Alice"));

        // when
        Map<String, Object> result = restApiClient.execute(
            "GET", "/users/1", null, null, null,
            "http://test", Map.of(), List.of());

        // then
        assertThat(result.get("success")).isEqualTo(true);
    }
}
```

### unsafe 메서드 차단 테스트

```java
@Test
void execute_postWithoutAllowList_blocked() {
    Map<String, Object> result = restApiClient.execute(
        "POST", "/users", null, null, Map.of("name", "John"),
        "http://test", Map.of(), List.of());  // 빈 allowList

    assertThat(result.get("success")).isEqualTo(false);
    assertThat(result.get("error").toString())
        .contains("not allowed");
}

@Test
void execute_postWithMatchingPattern_allowed() {
    when(/* ... mock setup ... */).thenReturn(Map.of("id", 1));

    Map<String, Object> result = restApiClient.execute(
        "POST", "/search", null, null, Map.of("q", "test"),
        "http://test", Map.of(), List.of("/search"));  // /search 허용

    assertThat(result.get("success")).isEqualTo(true);
}
```

## DuckDB 실제 사용 패턴

`DuckDbExecutor` 테스트는 **실제 DuckDB 인스턴스**를 사용합니다. 인메모리로 동작하므로 빠르고 부작용이 없습니다.

```java
class DuckDbExecutorTest {

    private final DuckDbExecutor executor = new DuckDbExecutor();

    @Test
    void extractTables_listResponse_usesName() {
        List<Map<String, Object>> data = List.of(
            Map.of("id", 1, "name", "Alice"),
            Map.of("id", 2, "name", "Bob")
        );

        var result = executor.extractTables(data, "users");
        assertThat(result.tables()).containsKey("users");
        assertThat(result.tables().get("users")).hasSize(2);
    }

    @Test
    void executeSql_simpleQuery_success() {
        Map<String, List<?>> data = Map.of(
            "users", List.of(
                Map.of("id", 1, "name", "Alice"),
                Map.of("id", 2, "name", "Bob")
            )
        );

        Map<String, Object> result = executor.executeSql(data,
            "SELECT * FROM users WHERE id = 1");

        assertThat(result.get("success")).isEqualTo(true);
        List<?> rows = (List<?>) result.get("result");
        assertThat(rows).hasSize(1);
    }

    @Test
    void truncateForContext_largeData_truncatesRows() {
        List<Map<String, Object>> data = generateLargeDataset(10000);

        var result = executor.truncateForContext(data, "data", 1000);
        assertThat(result.get("truncated")).isEqualTo(true);
        assertThat((int) result.get("showing")).isLessThan(10000);
    }
}
```

## 테스트 작성 팁

### 1. 순수 단위 테스트 우선

이 프로젝트는 Spring Context를 로딩하지 않는 순수 단위 테스트 위주입니다. 새로운 테스트도 이 패턴을 따르세요:

```java
class NewServiceTest {

    private final NewService service = new NewService(/* 의존성 직접 주입 */);

    @Test
    void methodName_scenario_expectedResult() {
        // given
        // when
        // then
    }
}
```

### 2. 테스트 이름 규칙

`메서드명_시나리오_기대결과` 형식을 따릅니다:

```java
@Test
void sanitize_nullInput_returnsRecipe() { ... }

@Test
void executeSql_invalidQuery_returnsError() { ... }

@Test
void suggestRecipes_noMatches_returnsEmptyList() { ... }
```

### 3. DuckDB는 실제 사용

`DuckDbExecutor`의 테스트는 모킹하지 않고 실제 DuckDB를 사용합니다. 인메모리 동작이므로 빠르고 실제 SQL 동작을 검증할 수 있습니다.

### 4. Virtual Threads와 테스트

Spring Boot의 Virtual Threads 설정은 테스트에 영향을 주지 않습니다. 테스트는 동기적으로 실행되며, ThreadLocal도 테스트 메서드 단위로 격리됩니다.
