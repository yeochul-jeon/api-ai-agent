## `ApiAgentProperties`: 환경변수 기반 설정

**파일 위치**: `src/main/java/com/apiagent/config/ApiAgentProperties.java`

Python 원본의 `config.py` (`pydantic-settings`)에 대응합니다. Java `record`로 구현되어 불변성을 보장합니다.

### 구현

```java
@ConfigurationProperties(prefix = "api-agent")
public record ApiAgentProperties(
        String mcpName,              // MCP 서버 이름
        String serviceName,          // 트레이싱 서비스 이름
        String modelName,            // LLM 모델명 (기본: gpt-4o)
        String reasoningEffort,      // 추론 노력 수준
        int maxAgentTurns,           // 최대 턴 수 (기본: 30)
        int maxResponseChars,        // 최대 응답 크기 (기본: 50000)
        int maxSchemaChars,          // 최대 스키마 크기 (기본: 32000)
        int maxPreviewRows,          // 미리보기 행 수 (기본: 10)
        int maxToolResponseChars,    // 도구 응답 크기 (기본: 32000)
        int maxPolls,                // 폴링 최대 횟수 (기본: 20)
        int defaultPollDelayMs,      // 폴링 간격 (기본: 3000ms)
        boolean debug,               // 디버그 모드
        String corsAllowedOrigins,   // CORS 도메인
        boolean enableRecipes,       // 레시피 활성화
        int recipeCacheSize          // 캐시 크기 (기본: 64)
) {
    // Canonical Constructor에서 기본값 설정
    public ApiAgentProperties {
        if (mcpName == null) mcpName = "API Agent";
        if (maxAgentTurns <= 0) maxAgentTurns = 30;
        // ...
    }

    public String mcpSlug() {
        // "API Agent" → "api_agent"
        return SLUG_PATTERN.matcher(mcpName.toLowerCase())
                .replaceAll("_").replaceAll("^_|_$", "");
    }
}
```

### Python 원본과의 차이

| Python (`config.py`) | Spring Boot (`ApiAgentProperties`) |
|----------------------|-----------------------------------|
| `pydantic-settings.BaseSettings` | `@ConfigurationProperties` record |
| `env_prefix="API_AGENT_"` | `prefix = "api-agent"` |
| `AliasChoices` (OPENAI_API_KEY) | `application.yml`에서 `${OPENAI_API_KEY:}` |
| `@computed_field MCP_SLUG` | `mcpSlug()` 메서드 |
| `settings = Settings()` (싱글턴) | Spring Bean 자동 주입 |

### 설정 바인딩 경로

```
환경변수: API_AGENT_MAX_AGENT_TURNS=50
    ↓
application.yml: max-agent-turns: ${API_AGENT_MAX_AGENT_TURNS:30}
    ↓
ApiAgentProperties: maxAgentTurns = 50
```

## `McpServerConfig`: MCP 도구 등록

**파일 위치**: `src/main/java/com/apiagent/config/McpServerConfig.java`

Python 원본의 `tools/__init__.py` (`register_all_tools`)에 대응합니다. Spring AI MCP Server가 자동으로 도구를 노출하도록 등록합니다.

```java
@Configuration
public class McpServerConfig {

    @Bean
    public MethodToolCallbackProvider mcpTools(McpToolProvider provider) {
        return MethodToolCallbackProvider.builder()
                .toolObjects(provider)  // @Tool 메서드 자동 탐색
                .build();
    }
}
```

`McpToolProvider`의 `@Tool` 어노테이션이 붙은 메서드가 MCP 도구로 자동 등록됩니다. Python 원본에서 `@mcp.tool(name="_query")`로 등록하던 것에 해당합니다.

## `RequestContextFilter` / `RequestContextHolder`: 요청 격리

### RequestContext (record)

**파일 위치**: `src/main/java/com/apiagent/context/RequestContext.java`

Python 원본의 `context.py` (`RequestContext` dataclass)에 대응합니다.

```java
public record RequestContext(
    String targetUrl,               // X-Target-URL
    String apiType,                 // X-API-Type: "graphql" | "rest"
    Map<String, String> targetHeaders, // X-Target-Headers (JSON)
    List<String> allowUnsafePaths,  // X-Allow-Unsafe-Paths (JSON 배열)
    String baseUrl,                 // X-Base-URL
    boolean includeResult,          // X-Include-Result
    List<String> pollPaths          // X-Poll-Paths (JSON 배열)
) {}
```

### RequestContextFilter (Servlet Filter)

**파일 위치**: `src/main/java/com/apiagent/context/RequestContextFilter.java`

Python 원본의 `context.py` (`get_request_context()`)에 대응합니다. FastMCP의 `get_http_headers()` 대신 Servlet `HttpServletRequest`에서 헤더를 추출합니다.

```java
@Component
@Order(1)
public class RequestContextFilter implements Filter {

    @Override
    public void doFilter(ServletRequest req, ServletResponse resp, FilterChain chain) {
        HttpServletRequest request = (HttpServletRequest) req;
        try {
            String targetUrl = request.getHeader("X-Target-URL");
            String apiType = request.getHeader("X-API-Type");
            Map<String, String> targetHeaders = parseJsonMap(
                request.getHeader("X-Target-Headers"));
            // ...

            RequestContext ctx = new RequestContext(
                targetUrl, apiType, targetHeaders, ...);
            RequestContextHolder.set(ctx);

            chain.doFilter(req, resp);
        } finally {
            RequestContextHolder.clear();  // 반드시 정리!
        }
    }
}
```

### RequestContextHolder (ThreadLocal)

**파일 위치**: `src/main/java/com/apiagent/context/RequestContextHolder.java`

Python 원본의 `ContextVar` 패턴에 대응합니다.

```java
public final class RequestContextHolder {
    private static final ThreadLocal<RequestContext> HOLDER = new ThreadLocal<>();

    public static void set(RequestContext ctx) { HOLDER.set(ctx); }
    public static RequestContext get() { return HOLDER.get(); }
    public static RequestContext require() {
        RequestContext ctx = HOLDER.get();
        if (ctx == null) {
            throw new IllegalStateException("RequestContext not set");
        }
        return ctx;
    }
    public static void clear() { HOLDER.remove(); }
}
```

> **중요**: `finally` 블록에서 반드시 `clear()`를 호출해야 합니다. Virtual Thread 풀에서 스레드가 재사용될 수 있으므로, 이전 요청의 컨텍스트가 남아있으면 데이터가 오염됩니다.

## `ToolNamingService`: 동적 도구 이름 변환

**파일 위치**: `src/main/java/com/apiagent/mcp/ToolNamingService.java`

Python 원본의 `middleware.py` (`DynamicToolNamingMiddleware`)와 `context.py`의 도구 이름 관련 함수에 대응합니다.

### 역할

하나의 MCP 서버가 여러 API를 동시에 지원할 수 있도록, URL에서 의미 있는 이름을 추출하여 도구 이름을 생성합니다.

### 주요 메서드

```java
@Service
public class ToolNamingService {

    // URL에서 도구 이름 접두사 생성
    public String getToolNamePrefix(String url) {
        // "https://flights-api-qa.internal.example.com/graphql"
        // → 호스트 파싱: "flights-api-qa.internal.example.com"
        // → 의미 없는 부분 제거: com, internal, api, qa
        // → 결과: "flights_example"
    }

    // 문자열을 snake_case로 변환
    public String toSnakeCase(String name) {
        // "API Agent" → "api_agent"
    }

    // API 이름 추출 (X-API-Name 헤더 우선)
    public String extractApiName(String apiName, String targetUrl) {
        // apiName이 있으면 사용, 없으면 URL에서 추출
    }

    // 도구 설명에 API 컨텍스트 주입
    public String injectApiContext(String description, String hostname, String apiType) {
        // "[flights.example.com GraphQL API] 원래 설명"
    }
}
```

### 제거되는 일반적인 단어

```java
Set<String> SKIP = Set.of(
    "com", "io", "is", "net", "org", "privatecloud",
    "qa", "dev", "internal", "api"
);
```

### Python 원본과의 차이

| Python (`middleware.py`) | Spring Boot (`ToolNamingService`) |
|--------------------------|----------------------------------|
| FastMCP Middleware 클래스 | Spring `@Service` |
| `on_list_tools` 훅 | 도구 등록 시 이름 변환 |
| `on_call_tool` 역변환 | 불필요 (Spring AI가 직접 매핑) |
| `_execute` 도구 포함 | `_query` 도구만 제공 |

## `DuckDbExecutor`: SQL 실행 엔진

**파일 위치**: `src/main/java/com/apiagent/executor/DuckDbExecutor.java`

Python 원본의 `executor.py`에 대응합니다.

### 주요 메서드

#### `extractTables(data, name)`

API 응답에서 DuckDB 테이블 데이터를 추출합니다:

```java
public TableExtraction extractTables(Object data, String name) {
    // 1. List인 경우: 직접 테이블로 사용
    //    [{"id": 1}, {"id": 2}] → "data" 테이블

    // 2. Map에 List 필드가 있는 경우: 첫 번째 List 사용
    //    {"users": [...], "total": 100} → "data" 테이블 (users 리스트)

    // 3. 단순 Map: 1행 테이블로 감싸기 + 스키마 정보 추출
    //    {"name": "John", "age": 30} → "data" 테이블 (1행)
}
```

#### `executeSql(data, query)`

JSON 데이터에 SQL을 실행합니다:

```java
public Map<String, Object> executeSql(Map<String, List<?>> data, String query) {
    // 1. DuckDB 연결 생성 (임시, 인메모리)
    // 2. JSON 데이터를 임시 파일로 저장
    // 3. read_json_auto()로 테이블 생성
    // 4. SQL 실행
    // 5. 결과를 Map 리스트로 변환
    // 6. 임시 파일 정리
}
```

> **왜 임시 파일?**: DuckDB의 `read_json_auto()`는 파일 경로를 받습니다. 메모리의 데이터를 직접 전달할 수 없어서 임시 JSON 파일을 거칩니다.

#### `truncateForContext(data, tableName, maxChars)`

LLM 컨텍스트 안전을 위한 데이터 잘림 (3단계):

```java
public Map<String, Object> truncateForContext(List<?> data, String tableName, int maxChars) {
    // 1단계: 전체 JSON이 maxChars 이하 → 그대로 반환
    // 2단계: 행 단위로 잘라서 포함 + 스키마 정보
    // 3단계: 단일 객체 → DuckDB 스키마 요약만 반환
}
```

#### `extractSchema(data, tableName)`

DuckDB의 `DESCRIBE` 명령으로 테이블 스키마를 추출합니다.

## `McpToolProvider`: MCP 도구 정의

**파일 위치**: `src/main/java/com/apiagent/mcp/McpToolProvider.java`

Python 원본의 `tools/query.py` (`_query` 도구)에 대응합니다.

### `_query` 메서드

```java
@Service
public class McpToolProvider {

    @Tool(name = "_query", description = "자연어로 API에 질문합니다")
    public String query(String question) {
        // 1. RequestContext 가져오기 (ThreadLocal)
        RequestContext ctx = RequestContextHolder.require();

        // 2. API 유형에 따라 에이전트 라우팅
        Map<String, Object> result;
        if ("graphql".equals(ctx.apiType())) {
            result = graphqlAgentService.process(question, ctx);
        } else {
            result = restAgentService.process(question, ctx);
        }

        // 3. 직접 반환 모드: CSV 형식으로 반환
        if (result.get("result") != null && result.get("data") == null) {
            return csvConverter.toCsv(result.get("result"));
        }

        // 4. 응답 잘림 적용
        return responseTruncator.truncate(objectMapper.writeValueAsString(result));
    }
}
```

### Python 원본과의 차이

| Python (`tools/query.py`) | Spring Boot (`McpToolProvider`) |
|---------------------------|-------------------------------|
| `@mcp.tool(name="_query")` | `@Tool(name = "_query")` |
| `get_request_context()` | `RequestContextHolder.require()` |
| `process_query()` (GraphQL) | `graphqlAgentService.process()` |
| `process_rest_query()` (REST) | `restAgentService.process()` |
| `to_csv()` 유틸리티 | `CsvConverter.toCsv()` |
| `consume_recipe_changes()` + `send_tool_list_changed()` | (Spring AI MCP에서 별도 처리) |
| `_execute` 도구 별도 존재 | `_query` 하나로 통합 |
