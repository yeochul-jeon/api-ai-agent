## 3. 주요 이슈 및 해결책

### 이슈 1: Gradle Kotlin DSL + Java 25 비호환

**증상:** Gradle 8.14.1에 번들된 Kotlin 컴파일러가 Java 25 버전 문자열을 파싱하지 못해 `JavaVersion.parse` 에러 발생.

**원인:** Kotlin 컴파일러 내부의 Java 버전 파서가 2자리 메이저 버전(25)을 처리하지 못함.

**해결:**
- `build.gradle.kts` → `build.gradle` (Groovy DSL)로 전환
- `.tool-versions`는 Java 21 유지 (Gradle JVM 실행용)
- `toolchain { languageVersion = JavaLanguageVersion.of(25) }`로 컴파일 타겟 지정

---

### 이슈 2: Lombok Java 25 호환

**증상:** Spring Boot BOM이 관리하는 기본 Lombok 버전에서 `sun.misc.Unsafe` 관련 `ExceptionInInitializerError` 발생.

**원인:** 이전 Lombok 버전이 내부적으로 `sun.misc.Unsafe`에 의존하며, Java 25에서 해당 API가 제한됨.

**해결:**
```groovy
compileOnly 'org.projectlombok:lombok:1.18.38'
annotationProcessor 'org.projectlombok:lombok:1.18.38'
```
Lombok 1.18.38을 명시적으로 지정하여 Java 25 호환성 확보.

---

### 이슈 3: Jackson 2 → 3 패키지 변경

**증상:** Spring Boot 4.x가 Jackson 3을 기본 사용하면서 기존 `com.fasterxml.jackson` import가 모두 컴파일 에러.

**변경 내용:**

| Before (Jackson 2) | After (Jackson 3) |
|--------------------|--------------------|
| `com.fasterxml.jackson.databind.ObjectMapper` | `tools.jackson.databind.ObjectMapper` |
| `com.fasterxml.jackson.databind.JsonNode` | `tools.jackson.databind.JsonNode` |
| `com.fasterxml.jackson.core.JsonProcessingException` | `tools.jackson.databind.JacksonException` |
| `new ObjectMapper(new YAMLFactory())` | `YAMLMapper.builder().build()` |

> **참고:** `jackson-annotations`만 여전히 `com.fasterxml.jackson.core` 그룹에 남아 있음.

---

### 이슈 4: Spring Boot 4.x API 변경

**변경 사항:**

| 항목 | Before (3.4.x) | After (4.0.3) |
|------|-----------------|---------------|
| HTTP 클라이언트 팩토리 | `SimpleClientHttpRequestFactory` | `JdkClientHttpRequestFactory` |
| TestRestTemplate 패키지 | `org.springframework.boot.test.web.client` | `org.springframework.boot.resttestclient` |
| TestRestTemplate 설정 | 자동 구성 | `@AutoConfigureTestRestTemplate` 명시 필요 |
| ToolResponseMessage 생성 | `new ToolResponseMessage(...)` | `ToolResponseMessage.builder()...build()` |

---

### 이슈 5: Spring AI 1.0 → 2.0 의존성 문제

**증상:** Spring Boot 4.x로 업그레이드 후 Spring AI 1.0.3에서 `RestClientAutoConfiguration` ClassNotFoundException 발생.

**원인:** Spring AI 1.0.x는 Spring Boot 3.x의 자동 구성 클래스에 의존하며, Boot 4.x에서 해당 클래스가 이동/제거됨.

**해결:**
- Spring AI 2.0.0-M3으로 동시 업그레이드 (Boot 4.x 호환 버전)
- milestone repository 추가 필요:
```groovy
repositories {
    maven { url 'https://repo.spring.io/milestone' }
}
```
