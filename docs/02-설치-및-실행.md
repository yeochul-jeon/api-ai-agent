## 사전 요구사항

| 항목 | 최소 버전 | 용도 |
|------|-----------|------|
| **Java** | 21+ (Corretto, Temurin 등) | 런타임 |
| **Gradle** | 8.x (Wrapper 포함) | 빌드 도구 |
| **Docker** (선택) | 최신 | 컨테이너 배포 시 사용 |

### Java 21 설치

> Java 21은 Virtual Threads, Record Patterns 등 최신 기능을 사용하기 위해 필수입니다.

```bash
# asdf (권장)
asdf install java corretto-21.0.6.7.1
asdf local java corretto-21.0.6.7.1

# SDKMAN
sdk install java 21.0.6-amzn
sdk use java 21.0.6-amzn

# Homebrew (macOS)
brew install --cask temurin@21

# 설치 확인
java -version
# openjdk version "21.0.6" ...
```

## 설치 단계

### 1. 저장소 클론

```bash
git clone <repository-url>
cd api-ai-agent
```

### 2. 빌드

```bash
# Gradle Wrapper로 빌드 (별도 Gradle 설치 불필요)
./gradlew build
```

이 명령은:
- 의존성을 자동 다운로드합니다
- 소스 코드를 컴파일합니다
- 테스트를 실행합니다
- JAR 파일을 `build/libs/`에 생성합니다

### 주요 의존성 목록

| 패키지 | 용도 |
|--------|------|
| `spring-boot-starter-web` (3.4.4) | 웹 서버 프레임워크 |
| `spring-ai-starter-mcp-server-webmvc` (1.0.3) | MCP 서버 프레임워크 |
| `spring-ai-starter-model-openai` (1.0.3) | AI ChatClient (OpenAI 호환) |
| `duckdb_jdbc` (1.2.1) | SQL 데이터 처리 엔진 |
| `fuzzywuzzy` (1.4.0) | 레시피 유사도 매칭 |
| `jackson-dataformat-yaml` | YAML 파싱 (OpenAPI 스펙) |
| `micrometer-tracing-bridge-otel` | OpenTelemetry 트레이싱 |

## 환경변수 설정

### 필수 환경변수

| 변수명 | 설명 | 예시 |
|--------|------|------|
| `OPENAI_API_KEY` | OpenAI API 키 | `sk-proj-abc123...` |

### 선택 환경변수

| 변수명 | 기본값 | 설명 |
|--------|--------|------|
| `OPENAI_BASE_URL` | `https://api.openai.com/v1` | OpenAI 호환 API 엔드포인트 (Azure, 로컬 LLM 등) |
| `API_AGENT_MODEL_NAME` | `gpt-4o` | 사용할 LLM 모델 이름 |
| `API_AGENT_REASONING_EFFORT` | (빈 문자열) | 추론 노력 수준: `low`, `medium`, `high` |
| `API_AGENT_PORT` | `3000` | 서버 포트 |
| `API_AGENT_DEBUG` | `false` | 디버그 모드 (상세 로그) |
| `API_AGENT_CORS_ALLOWED_ORIGINS` | `*` | CORS 허용 도메인 |
| `API_AGENT_ENABLE_RECIPES` | `true` | 레시피 캐싱 활성화 |
| `API_AGENT_RECIPE_CACHE_SIZE` | `64` | 레시피 캐시 최대 항목 수 |
| `API_AGENT_MAX_AGENT_TURNS` | `30` | 에이전트 최대 턴(도구 호출) 수 |

> 모든 설정은 `application.yml`에서 환경변수로 매핑됩니다. `API_AGENT_` 접두사 규칙을 따릅니다.

## 실행 방법

### 방법 1: Gradle bootRun (개발용)

```bash
OPENAI_API_KEY=sk-proj-your-key ./gradlew bootRun
```

서버가 `http://localhost:3000/mcp`에서 시작됩니다.

### 방법 2: JAR 직접 실행

```bash
# 먼저 빌드
./gradlew build -x test

# JAR 실행
OPENAI_API_KEY=sk-proj-your-key java \
  --enable-preview \
  -Dspring.threads.virtual.enabled=true \
  -jar build/libs/api-ai-agent-0.1.0.jar
```

### 방법 3: Docker

```bash
# 먼저 JAR 빌드
./gradlew build -x test

# 이미지 빌드
docker build -t api-agent .

# 실행
docker run -p 3000:3000 \
  -e OPENAI_API_KEY="sk-proj-your-key" \
  api-agent
```

Docker 이미지 특징:
- `eclipse-temurin:21-jre-alpine` 기반 (경량)
- ZGC 가비지 컬렉터 사용 (`-XX:+UseZGC`)
- Virtual Threads 활성화
- 메모리 제한 512MB

### 방법 4: Docker Compose

```bash
# .env 파일에 OPENAI_API_KEY 설정 후
docker compose up -d
```

`docker-compose.yml`은 다음을 자동 설정합니다:
- 포트 매핑 (3000:3000)
- 환경변수 주입
- 헬스체크 (30초 간격, `/actuator/health`)
- 자동 재시작

## 헬스체크 확인

서버가 정상 작동 중인지 확인합니다:

```bash
# Spring Boot Actuator 헬스체크
curl http://localhost:3000/actuator/health
```

정상 응답:
```json
{"status": "UP"}
```

### MCP 엔드포인트

MCP 클라이언트(Claude Desktop 등)에서 연결할 때 사용하는 URL:

```
http://localhost:3000/mcp
```

### MCP 클라이언트 설정 예시

Claude Desktop이나 다른 MCP 클라이언트에서 연결할 때, HTTP 헤더를 통해 대상 API를 지정합니다:

```json
{
  "mcpServers": {
    "my-api": {
      "url": "http://localhost:3000/mcp",
      "headers": {
        "X-Target-URL": "https://api.example.com/graphql",
        "X-API-Type": "graphql",
        "X-Target-Headers": "{\"Authorization\": \"Bearer your-token\"}"
      }
    }
  }
}
```
